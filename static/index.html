Blockchain Verification

        

        

          How tamper detection works
          
When a batch is completed, Forge creates a SHA-256 hash of the full electronic batch record and anchors only that hash with a transaction id. If any stored batch data changes later, the recalculated hash no longer matches the anchored hash.

          
The blockchain evidence is stored in the MES anchor table as entity type, entity id, stored hash, tx id, backend, and anchor metadata. The original record stays in the MES database, while the anchored hash acts as an immutable proof point.

          
Use Verify Anchor to compare the current record to the anchored blockchain hash, or export the PDF to capture the verification details in the eBR.

Batch To Proof Flow

          

            

              1. Operator + Equipment
              Actions, telemetry, signatures, and material usage enter the MES.
            

            
->

            

              2. eBR + Genealogy
              Batch events, recipe context, and genealogy assemble into one verifiable record.
            

            
->

            

              3. SHA-256 Hash
              The completed batch record is converted into a canonical hash.
            

            
->

            

              4. Blockchain Anchor
              Only the hash and tx id are anchored, so later changes become detectable.
            

            
->

            

              5. Verification + Export
              Forge recomputes the hash, compares it to the stored anchor, and exports the result in the eBR PDF.
