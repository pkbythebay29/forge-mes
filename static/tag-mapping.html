Industrial Tag Mapping

How OPC UA and MQTT source tags map into batch execution

Back to Operator Workspace Open Technical Guide

What Connected Means

When you click connect for OPC UA or MQTT in Forge, the demonstrator marks that driver endpoint as active and exposes the tag mappings required to translate industrial source data into MES fields.

In a production deployment, this is where browse, subscribe, and value conversion logic would begin streaming source data into the batch, material, or equipment context.

Manual OPC UA Server Configuration

To configure an OPC UA source manually, provide:

Server endpoint: opc.tcp://your-server:4840

Security mode: None / Sign / SignAndEncrypt

Namespace index: for example 2

Authentication: anonymous or username/password

In this demonstrator, the driver stores the endpoint and tag mapping definition so the MES knows which source nodes should drive which batch fields.

OPC UA Mapping Examples

ns=2;s=Batch.Id -> batch.batch_number

ns=2;s=Batch.Status -> batch.status

ns=2;s=Batch.StartCommand -> batch.start_command

ns=2;s=Batch.StopCommand -> batch.stop_command

ns=2;s=Batch.RecipeName -> recipe.name

ns=2;s=Process.ActualQuantity -> batch.actual_quantity

ns=2;s=Process.StepIndex -> batch.current_step

ns=2;s=Equipment.Mixer001.Runtime -> equipment.runtime_minutes

MQTT Mapping Examples

forge/line1/batch/id -> batch.batch_number

forge/line1/batch/start -> batch.start_command

forge/line1/batch/complete -> batch.complete_command

forge/line1/batch/step -> batch.current_step

forge/line1/equipment/mix001/status -> equipment.status

forge/line1/equipment/mix001/counts/good -> equipment.good_count

How To Add Tag Maps Manually

Define each mapping with five parts:

source_tag: the OPC UA node id or MQTT topic

mes_field: the target MES field such as batch.status

type: string, integer, float, or boolean

direction: read, write, subscribe, or publish

meaning: what the tag controls, for example batch start, active step, actual quantity, or equipment status

Why Tag Mapping Matters

MES workflows become much easier to trust when the bridge between industrial data and MES fields is explicit. Tag mapping makes it clear which source value updates which batch field, which equipment metric, and which part of the electronic batch record.

Live Driver Mapping API

GET /drivers/opcua/tag-map

These endpoints expose the current tag mapping definitions so an engineer can inspect how live industrial data would feed the MES record.
